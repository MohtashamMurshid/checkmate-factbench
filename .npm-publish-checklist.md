# NPM Publishing Checklist

## Before Publishing

1. **Update package.json:**
   - [ ] Set correct `version` (follow semver: MAJOR.MINOR.PATCH)
   - [ ] Fill in `author` field
   - [ ] Add `repository.url` with your GitHub/git URL
   - [ ] Add relevant `keywords` for discoverability
   - [ ] Update `description` if needed

2. **Test the build:**
   ```bash
   bun run build
   ```

3. **Verify dist folder contains:**
   - `index.js` (ESM)
   - `index.cjs` (CommonJS)
   - `index.d.ts` (TypeScript declarations)
   - `index.d.ts.map` (source maps)

4. **Test locally (optional):**
   ```bash
   npm pack
   # This creates a .tgz file you can test
   ```

## Publishing

1. **Login to npm:**
   ```bash
   npm login
   ```

2. **Publish:**
   ```bash
   npm publish
   ```
   
   For scoped packages or first-time publish:
   ```bash
   npm publish --access public
   ```

3. **Verify publication:**
   - Check npmjs.com/package/checkmate-factbench
   - Test installation: `npm install checkmate-factbench`

## After Publishing

- [ ] Create a git tag: `git tag v0.1.0 && git push --tags`
- [ ] Update CHANGELOG.md (if you create one)
- [ ] Announce the release

## Notes

- The `prepublishOnly` script automatically runs `bun run build` before publishing
- Only files listed in `package.json` > `files` will be published
- Source TypeScript files are excluded via `.npmignore`
